Baker can convert, process, extract or store records (structured data), applying whatever transformation between input and output through easy-to-write filters. It is fully parallel and maximizes usage of both CPU-bound and I/O bound pipelines. Baker is an open source project made in NextRoll, inc. and it's completely written in Go {{% /blocks/lead %}} {{< blocks/section color="primary" >}} {{% blocks/feature icon="fas fa-link" title="Modular pipeline" %}} Baker uses modular components in a 4-levels stack: inputs, filters, outputs and uploads. Each component of a level can connect with any other from the previous and the next one. {{% /blocks/feature %}} {{% blocks/feature icon="fas fa-shipping-fast" title="High performances" %}} Baker is fully parallel and maximizes usage of both CPU-bound and I/O bound pipelines. On a c5.2xlarge EC2 instance, Baker managed to read zstandard records from S3, uncompress them and apply a basic filtering logic, compressing them back on local files with zstandard using ~90% of capacity of each vCPU (8 in total) and ~3.5GB of RAM, reading and writing ~178000 records per second. {{% /blocks/feature %}} {{% blocks/feature icon="fas fa-puzzle-piece" title="Extendable" %}} With Baker it's super easy to write new components or process any kind of structured data. Read the How-tos pages in the documentation to learn how to do it. {{% /blocks/feature %}} {{< /blocks/section >}}
