Blazing fast and open-source data processor

Baker pipelines can fetch, transform and store records in a flash thanks to a high-performant and fully-parallel implementation.

Baker is open-source and written in Go by NextRoll, inc.

{{< blocks/section color="primary" >}} {{% blocks/feature icon="fas fa-link" title="Modular pipeline" %}} Baker pipelines are assembled using up to 4 kinds of components: inputs, filters, outputs and uploads. They are fully modular: any component can be replaced with another of the same kind. Available components include many AWS services (S3, Kinesis, DynamoDB, SQS) as well as websocket, TCP, local disk, etc.

Check out the full list »

{{% /blocks/feature %}} {{% blocks/feature icon="fas fa-shipping-fast" title="High performances" %}} Baker is fully parallel and maximizes the usage of both CPU-bound and I/O bound pipelines. On an AWS EC2 instance of size c5.2xlarge, Baker can process compressed records read from S3 and written to local disk, with basic filtering, using ~90% of CPU and ~3.5GB of RAM, for an average R/W speed of 700MB/sec.

More performances examples »

{{% /blocks/feature %}} {{% blocks/feature icon="fas fa-puzzle-piece" title="Extendable" %}} With Baker it is super easy to write your own components and process any kind of structured data. Baker processes records of structured data that can be read, filtered and written using many available components written in Go. If you can't find a component that suits your needs, it's very easy to create a new one.
